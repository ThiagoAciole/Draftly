# AI Context

Esta pasta contem a memoria tecnica resumida do Draftly.

Ela foi criada para ajudar desenvolvedores e agentes de IA a entenderem rapidamente a arquitetura, os padroes, as features e os pontos de atencao do projeto sem reler todo o codigo.

## Ordem recomendada de leitura

1. `context-pack.md`
2. `project-overview.md`
3. `architecture.md`
4. `features.md`
5. `agent-instructions.md`

## Arquivos

- `context-pack.md`: resumo compacto para colar em qualquer IA.
- `project-overview.md`: visao geral do produto e dos modulos.
- `architecture.md`: camadas, responsabilidades e acoplamentos.
- `folder-structure.md`: estrutura de pastas real.
- `tech-stack.md`: tecnologias, bibliotecas e scripts.
- `features.md`: funcionalidades existentes mapeadas no codigo.
- `code-patterns.md`: padroes de codigo e testes.
- `ui-patterns.md`: padroes visuais, layout, temas e estados.
- `data-flow.md`: fluxo de dados entre UI, estado, services e Rust.
- `api-and-integrations.md`: comandos Tauri e integracoes do sistema.
- `state-management.md`: estado local, global e persistencia.
- `domain-rules.md`: regras de negocio e comportamento.
- `decisions.md`: decisoes tecnicas identificadas.
- `current-state.md`: estado atual e riscos.
- `changelog.md`: historico desta memoria.
- `agent-instructions.md`: instrucoes praticas para code agents.
- `update-rules.md`: regras para manter a memoria atualizada.

Atualize esta pasta sempre que uma mudanca alterar arquitetura, features, APIs, estado, regras de negocio, padroes de codigo ou fluxos importantes.
